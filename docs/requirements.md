## 0. 术语约定

- **帧**:除非特别说明,均指**软件节流后实际送入 YOLO 推理的帧**(不是相机原始 60 fps 输出帧)。
- **皮带固定坐标系(belt 系)**:以**任务启动瞬间**的皮带位置为原点,物体静止在皮带上时坐标保持恒定,单位 mm;`yb` 沿皮带运动方向递增,`xb` 与运动方向正交。
- **最近边缘**:图像坐标中,物体沿皮带运动方向**即将离开视野**的那条边。
- **t_capture**:相机采集到该帧瞬间的系统单调时钟戳;一旦打上,**整条链路(YOLO/Tracker/Dispatcher/Board)都不允许重新赋值**,所有"物体当前位置"由 `speed × (t_now - t_capture)` 推出。

## 1. 项目目标与范围

### 1.1 一句话定义

在 RK3588 工控机上做一台**光选分拣机的软件大脑**:相机不间断采图 → YOLOv8-Seg 识别皮带上的物体 → 跨帧合并成"同一颗物体" → 在物体到达分拣器之前,把它的位置/类别交给**喷阀阵列**或**机械臂(本期 stub)**完成分拣动作。

### 1.2 物理形态

```
┌──────────┐                      ┌──────────────┐
│ 海康 GigE │ ────── 看 ──────▶  │  皮带 + 编码器│ ──── 物体流向 ────▶
│   相机    │                      └──────────────┘                    │
└──────────┘                                                            │
                                                                        ▼
                              ┌────────────────┐    ┌──────────────────────┐
                              │ 喷阀阵列       │ 或 │ 机械臂 A/B(本期 stub)│
                              │ 72 通道 × 8 板 │    │ 仅算坐标 + 写日志     │
                              └────────────────┘    └──────────────────────┘
```

- **主机**:1 台 RK3588 工控机(aarch64,Qt5,RKNN 2.x)
- **采图**:1 台海康 GigE 相机(MVS SDK,硬件帧率 60 fps,实际推理 fps 由软件节流决定)
- **执行机构**:RS485 串口阀板(8 块板 × 9 通道 = 72 通道)、双机械臂 A/B(本期不下发命令)
- **反馈**:同一根串口上的编码器帧,500 ms 周期采样皮带瞬时速度

### 1.3 本期做 / 不做

| 类别 | 本期 |
|---|---|
| **做** | 相机采图 + 软件节流;YOLO 推理(常驻模型) + box/mask/类别;belt 系 mask-IoU 跨帧追踪 + 已分拣抑制池;**喷阀分拣完整闭环**;前端启停 / 品类选择;识别结果实时可视化 |
| **不做** | 机械臂实际抓取动作(只算 A/B 坐标系下的位置 + 写日志/CSV,**不下发 TCP 命令**) |
| **保留 UI 但不接业务** | 视觉识别页的"颜色 / 标签 / 形态 / 品相" 4 组按钮(为后续多维度模型预留) |

## 2. 整体处理流程

任何一颗物体从"被相机看到"到"被分拣"都走以下一条流水线;横向贯穿"编码器实时速度"和"任务生命周期"两条支线。

```
┌──────┐  软件节流   ┌─────┐  conf+NMS  ┌────────┐  belt系mask-IoU ┌──────┐  按 mode  ┌──────┐
│ 相机 │ ──────────▶ │ YOLO│ ─────────▶ │ Tracker│ ─────────────▶  │触发  │ ────────▶ │ 喷阀 │ → 串口
│采图  │  打 t_cap   │推理 │  过滤      │跨帧合并│  抑制池防二次   │分拣  │           └──────┘
└──────┘             └─────┘            └────────┘                  └──────┘ ────────▶ ┌──────┐
   ▲                                       ▲                          │                │机械臂│ → 仅日志
   │                                       │                          │                │ stub │
   │                                       │ speed                    │                └──────┘
   │                                  ┌────┴─────┐                    │
   └───── fps / 距离 / 品类 ◀────────│ Encoder  │ ◀────── 编码器响应──┘
                                      │ 500ms 采│
                                      └──────────┘
                                            │
                       ┌────────────────────▼────────────────────┐
                       │    控制面:启停 / 参数配置 / 告警状态栏  │
                       └─────────────────────────────────────────┘
```

**全局不变量**(任何模块都不能违反):

1. **时间一条线**:`t_capture` 由相机一次性打戳,跨模块只读不写。所有"物体现在在哪"都通过 `t_now - t_capture` × 速度推出来,而不是各自补"经验延迟"。
2. **位置一条线**:几何计算统一在 belt 系(mm)。像素 → mm 只在 Tracker 入口做一次,出 Tracker 之后全程 belt 系。
3. **停机硬清零**:一次 stop 必须把追踪列表 / 已分拣池 / 待喷队列 / 串口未发命令全部清掉,串口不再下发,所有控件恢复可用。
4. **启动后第一帧丢弃**:任务启动瞬间已经在视野中(可能已部分越过镜头)的物体一律忽略,避免坐标原点不齐的"在途物体"被误分拣。

## 3. 各模块要实现什么

按"采图 → 识别 → 追踪 → 派发 → 执行"+ 横向"控制面 / 配置"组织。每个模块给出**职责定位**和**关键约束**;具体的需求条目见第 4 节。

### 3.1 控制面(前端 + Session 状态机)

**定位**:整套系统的唯一人机入口。负责启停、运行态可视化、品类选择、告警汇总。

**职责**:

- 把 `config.ini` 读成一份运行时配置快照,任务启动瞬间一次性下发给所有 worker,运行中**不再回读文件**。
- 启动前做硬检查 + 软检查;运行中锁定品类勾选;停止后解锁。
- 把所有 worker 的告警汇集到状态栏 + 日志文件。
- 实时显示 YOLO 识别覆盖层(box / mask / 品类标签 / 置信度)。

**关键约束**:

- 启动前**强制项**(一项不过就不允许启动):相机已连接。
- 启动前**软项**(失败仅告警仍允许启动):至少勾选 1 个目标品类。
- 分拣器模式和运行参数来自 `config.ini`,本期不在前端 UI 暴露。
- "颜色 / 标签 / 形态 / 品相" 4 组按钮 UI 可见,但**勾选与否对业务零影响**。

### 3.2 采图(CameraWorker)

**定位**:对接海康 MVS SDK,把相机原始 60 fps 的输出节流到业务真正需要的频率,并给放行的每一帧打上时间戳。

**职责**:

- 软件节流:按 `1000 / soft_fps ms` 的间隔放行帧,默认 `soft_fps = 2`(每 500 ms 放行一帧)。
- 在**采图瞬间**记录 `t_capture`(系统单调时钟,ms),贯穿后续所有流程。
- 反压(单元素队列):上一帧仍在推理时,新帧直接丢弃 + 累计告警。
- 持久化:可选地把"放行的帧"按比例采样保存到磁盘(F16)。

**关键约束**:

- 节流丢弃的帧不进入任何后续流程。
- 反压丢帧需可观测(日志计数),不静默丢。
- 保存原图的对象是**节流后的帧**,不是相机原始帧。

### 3.3 识别(YoloWorker + RKNN session)

**定位**:常驻一个 YOLOv8-Seg session,把每帧 RGB → 物体的 box/mask/class/conf。

**职责**:

- **进程启动时一次性加载模型**,常驻直到进程退出;模型路径由配置文件指定,**不在前端配置、不支持热切换**,改了必须重启进程。
- 每帧推理后做两步过滤:`conf_threshold` + 按类 NMS(`nms_threshold`),阈值都是配置项,不再硬编码。
- 输出帧级时间戳 `tCaptureMs` / `tInferDoneMs`,每个目标携带 `class_id`、`confidence`、`bbox`、`mask`、`center_px`。
- 渲染可视化叠加图(box + mask 半透明 + 中文品类名 + 置信度)发给前端。

**关键约束**:

- 模型只在进程启动 / 模型路径变更时初始化一次;**每帧不重复 init/release**。
- 类别标签来自 ini 中配置的 `class_btn/N/name`,未配置的 `class_id` 回落到数字 id。
- 某帧推理结果为空时,UI 覆盖层必须**立即清空**,不能保留上一帧的覆盖层。
- 后处理实现要求做"分桶 top-K + 矩形 IoU NMS,只对保留项算 mask",避免对全部候选都算 mask。

### 3.4 跨帧追踪(TrackerWorker)

**定位**:把每帧独立的 YOLO 检测结果合并成"同一颗物体"在 belt 系上的轨迹,并在合适时机触发分拣。

**职责**:

- **像素 → belt 系栅格**:每个新检测按物理比例 (`real_length_mm / W_px`, `real_width_mm / H_px`) 转 mm,再按 `mask_raster_mm_per_px`(默认 2)栅格化。
- **皮带位移外推**:与历史追踪关联前,把历史 bbox 按 `speed × (t_now - tCapture)` 外推到本帧的 `t_capture` 时刻;计算只在外推时加位移,栅格化阶段不重复加(避免双位移 bug)。
- **关联**:detection × active 全配对,IoU ≥ 阈值进候选;按 IoU 降序贪心匹配(本期不用匈牙利)。
- **合并**:命中的 mask 做 union 融合;`updateCount += 1`,`missCount = 0`;`classAreaAcc[classId] += area`。
- **新物体**:未匹配的检测先查"已分拣抑制池",命中则与 ghost 做 union 融合并刷新 ghost 时间戳,**不新建追踪**;否则才新建。
- **触发分拣**:`miss ≥ X` 或 `update ≥ Y` 满足任一即触发。最终类别取 `classAreaAcc` 最大者;若不在前端勾选品类内,**直接丢弃不触发分拣**。
- **抑制池清理**:ghost 的 `yb_trail` 越过分拣线 + `dispatched_pool_clear_distance_mm`(默认 200 mm)时移除。

**关键约束**:

- 多对一冲突时**贪心 IoU 最大者关联**,落选检测进入下一轮(可与其他追踪匹配,否则视为新物体)。
- 启动后第一帧的所有检测进入 ghost 抑制池,不入 active、不触发分拣,从第二帧开始正常处理。
- 触发分拣后,该物体**不能再触发第二次**(由抑制池保证)。

### 3.5 派发(Dispatcher)

**定位**:把 Tracker 输出的"分拣任务"翻译成执行机构能直接吃的指令——喷阀的时间轴 / 机械臂的坐标。

**职责(喷阀路径)**:

- **预计算为主、实时重算为辅**:物体进入待喷队列瞬间,按当前速度一次性算出整条 `ValvePulse` 时间轴(每条带 `tOpenMs / tCloseMs / boardId / channelMask`),交给 BoardWorker 在 5 ms tick 上真正下发。
- **头部留白**:从前沿开始累积非零像素到 `1/a × totalArea`(默认 1/8)时才开阀,补偿气流响应延迟。
- **尾部不留白**:物体后沿完全越过喷阀线后才关阀。
- **滚动投影追踪**:在 `[t_open_start, t_tail_arrive]` 区间内,以 `open_duration_ms`(默认 50)为步长重新投影"喷阀线对应物体哪一行像素 → 横向区间 → 通道集合 → boardId × bitmask"。
- **合并相邻命令**:同一 board 上 `tOpenMs` 间距小于 `min_cmd_interval_ms` 的相邻 pulse,位图 OR 合并、`tClose` 取大。
- **速度重算**:速度变化超过 `valve_speed_recalc_pct`(默认 20%)时,对每条 pending,先 `cancelPulses(trackId)` 再以新速度重算并 `enqueuePulses`,**整体取消整体重算**,不做 per-pulse 差分。

**职责(机械臂路径)**:

- 仅按标定参数计算物体在机械臂 A 坐标系、B 坐标系下的位置,写 `arm_stub.csv`(按日期滚动)+ 落日志。
- **不下发 TCP**;`tcpforrobot` 模块本期不在分拣链路上。

**关键约束**:

- Dispatcher 自己**不写串口**;时间轴 100% 交给 BoardWorker 执行。
- 头部留白后已经过线(`t_open_start > t_tail_arrive`)→ 物体太小,产生空脉冲列表 + 告警 `object_too_small`,**不入 pending**。
- 喷阀硬件参数(总通道、每板通道数、横向区间映射)只从 `config.ini` 读,**不在前端暴露**。
- 阀板地址 / 通道按 `(channel_idx - 1) // channels_per_board + 1 = boardId`、`bit = (channel_idx - 1) % channels_per_board` 严格对齐,不允许临时调换。

### 3.6 执行(BoardWorker / RS485 单总线)

**定位**:RS485 串口的唯一持有者。同一根串口上承担两件互斥的事:发阀控帧 + 问编码器。

**职责**:

- 串口打开 / 重连(`/dev/ttyUSB0` 失败回退 `ttyUSB1`,115200 8N1)。
- **5 ms tick** 扫描 `m_pending`,按 `tOpenMs / tCloseMs ≤ now` 真正下发开/关阀帧;全部 close 完成后从 pending 移除该 trackId。
- **500 ms tick** 发编码器请求帧,读 11 字节响应,按口径 `speed_m_per_min = raw × encoder_raw_to_m_per_min`(默认 0.502)→ `speed_mm_per_ms = m/s = m_per_min / 60` 解析(raw 是板卡内部采样好的"瞬时转速代理",不是累计脉冲也不是窗口增量),`emit speedSample` 给 Tracker / Dispatcher / UI。
- `onSessionStop` 时把所有"已 open 未 close"的 pulse 强制下发关阀帧后清空,保证停机后阀门不残留长开。
- `onCancelPulses(trackId)` 只取消"未开"的 pulse,**保留"已开未关"**,避免漏关阀。

**关键约束**:

- 全部串口 IO 经过单一互斥锁 `m_busMutex`,锁粒度 = 单帧(几百 μs),**不跨开/关持锁**。
- `onValveTick` 与 `onEncoderTick` 都在 BoardWorker 自己的事件循环上,天然不会重入。
- 串口协议规格不允许改,详见第 5 节硬件协议。

### 3.7 配置与持久化

**定位**:把所有可调参数集中到一个 `config.ini`,worker 持快照运行,改了文件必须重启进程才生效。

**配置必须覆盖**(详细 schema 见 `docs/design.md §5`):

- 相机:IP、硬件帧率、`y_flip`、`real_length_mm` / `real_width_mm`
- 模型 & YOLO:`path`、`input_size`、`conf_threshold`、`nms_threshold`、品类按钮 → `class_id` 映射
- Pipeline:`soft_fps`、`iou_threshold`、`miss_frames_x`、`update_frames_y`、`dispatched_pool_clear_distance_mm`、`mask_raster_mm_per_px`、`tick_interval_ms`
- 皮带 / 编码器:`nominal_speed_m_s` (fallback)、`encoder_raw_to_m_per_min`、`encoder_request_interval_ms`
- 分拣器:`mode`、`arm_distance_mm`、`valve_distance_mm`
- 喷阀:`total_channels`、`boards`、`channels_per_board`、`x_min_mm` / `x_max_mm`、`head_skip_ratio`、`open_duration_ms`、`min_cmd_interval_ms`、`speed_recalc_threshold_pct`
- 机械臂 stub 标定:A/B 各自的 `origin_x_mm / origin_y_mm / x_sign / y_sign`
- 持久化:`save_raw`、`raw_sample_ratio`、`save_result`、`save_dir`、`arm_stub_csv`

**持久化策略**:

- 3 个开关:**保存原图**(总开关) / **原图采样比例**(0.0–1.0) / **保存识别结果图**(仅当原图被保存才生效)。
- "保存原图"的对象是**节流后送入推理链的帧**,不是相机原始帧。
- "保存识别结果图":只有该帧的原图被保存时才保存对应的结果图。

## 4. 本期具体需求清单

> 以下 F 编号沿用 v1.1,作为本期开发与验收的可追溯锚点。已按第 3 节模块归位。同一条需求只列在它的主属模块下,不重复罗列。

### 4.1 控制面

#### F1 分拣器选择
- 分拣器模式由 `config.ini` 的 `[sorter] mode` 配置,取值为 `arm` 或 `valve`。
- 本期前端 UI 不暴露分拣器单选控件,启动时按配置快照下发给 Dispatcher。
- 机械臂模式本期仅 stub(日志 + CSV),不实际下发 TCP 命令;喷阀模式走 RS485 阀板。

#### F2 运行参数配置

| 参数 | 单位 | 说明 |
|---|---|---|
| 皮带速度(标称值) | m/s | 仅作 fallback,实际流程使用编码器实时值 |
| 相机拍照频率 | fps | 软件节流间隔 = 1000 / fps ms |
| 机械臂距图像最近边缘垂直距离 | mm | — |
| 喷阀距图像最近边缘垂直距离 | mm | — |

- 上述运行参数均通过 `config.ini` 配置,本期前端 UI 不暴露输入控件。
- 修改 `config.ini` 后必须重启进程生效;运行中 worker 只使用启动时下发的配置快照。

#### F3 目标品类选择
- 视觉识别页保留"品类"按钮组,选项从配置文件动态加载。
- 按钮与 YOLO 模型 `class_id` 建立映射(`class_btn/N/name` 与 `class_btn/N/id`)。
- 勾选的品类集合作为识别结果的**筛选条件**——不在勾选集合内的物体不触发分拣。
- 其它 4 组按钮(颜色/标签/形态/品相)**UI 可见、勾选与否不影响任何业务逻辑**。

#### F4 启动 / 停止

- **启动前强制检查**(失败禁止启动,提示原因):
  1. 相机已连接。
- **启动前软检查**(失败仅告警仍允许启动):
  1. 至少勾选 1 个目标品类。
- **运行中**:
  - 品类勾选在运行中**只读禁用**。
  - 识别可视化实时更新。
  - 运行中相机掉线**不再强制检查**,仅告警。
- **启动瞬间"在途物体"策略**:任务启动后**第一帧**推理的所有检测结果进入"已分拣抑制池"(从外部行为看等同丢弃),不形成正常追踪、不触发任何分拣;后续帧若继续检测到同一在途物体,仍由抑制池拦截,避免误分拣。
- **停止行为**:硬停 ——立即终止所有线程的处理路径、清空追踪列表 / 已分拣池 / 待喷队列、已预计算未下发的阀控命令一律作废、串口不再发任何命令。停止后重新启用所有参数控件。

#### F5 实时识别结果可视化
- 画面内容:**只绘制 YOLO 识别结果的 box、mask、类别标签**。
- 类别标签显示为人类可读的品类名称(对应 F3 中配置的按钮名称),可附带置信度。
- 不绘制 `track_id`、不绘制历史轨迹。
- 刷新频率 = 识别完成频率(异步推送,不阻塞推理链)。
- **空检测处理**:某帧推理结果为空时,立即清空画面上的 box/mask 覆盖层,只显示原始画面,不保留上一帧覆盖层。

#### F6 状态与告警输出
- 告警同时输出到:**日志文件** + **状态栏**;状态栏 1 秒内合并展示,避免刷屏。
- 等级:ERROR(阻断启动,如相机/模型加载失败)/ WARN(任务继续,如编码器读不到、丢帧、阀预计算物体过小)/ INFO(任务启停、SortTask 触发、ArmStub 派发)。

#### F7 其它界面约定
- 窗口分辨率与现有布局保持一致,本期不调整。
- 现有"机械臂分段延时标定面板"整块废弃。

### 4.2 采图

#### F9 相机采图与软件节流
- 相机硬件帧率保持现状(60 fps),本期不调整相机侧帧率。
- 软件节流:按 `1000 / fps ms` 的间隔送帧进入推理链。
- **默认 `fps = 2`**(每 500 ms 进入推理一次)。
- `fps` 通过配置文件调整,上限不超过推理链实测吞吐(见 F10 反压)。
- 节流丢弃的相机原始帧不进入后续任何流程。
- 每张"放行"的帧必须在**相机采图瞬间**打上 `t_capture`(基于系统单调时钟),该时间戳贯穿后续所有流程,不允许任何下游重新赋值。

#### F10 帧处理反压机制
- 推理链**单元素队列**(不做并行推理)。
- 新帧到达时若上一帧仍在处理:**丢弃新帧** + 告警(日志 + 状态栏)。
- 丢帧需可观测:每次丢帧累计计数器,达到阈值后窗口告警,**不静默**。

### 4.3 识别

#### F8 模型常驻
- YOLO 模型在**程序启动时**加载一次、常驻内存。
- **每次推理不重新 init / release**;Session 反复 start/stop 不重新加载模型。
- 模型路径通过配置文件指定,**不支持前端配置、不支持热切换**;修改后必须重启进程生效。

#### F11 YOLO 推理输出内容

**两步过滤(阈值由硬编码改为配置项)**:

- **置信度阈值** `conf_threshold`(默认 0.25):候选框置信度低于此值直接丢弃。
- **NMS IoU 阈值** `nms_threshold`(默认 0.45):同类候选框之间 IoU 大于此值的,保留置信度最高者(按类分桶 + top-K + 矩形 IoU NMS)。

过滤后的结果才进入跨帧追踪(F13)。

**帧级输出至少携带**:

- 采图时间戳 `tCaptureMs`(来自 F9,`qint64`,毫秒,进程级单调时钟)
- 推理完成时间戳 `tInferDoneMs`(`qint64`,毫秒,进程级单调时钟)
- 原图尺寸 `imgWidthPx` / `imgHeightPx`
- 目标列表 `objs`

**每个识别目标至少携带**:

- 类别 `class_id`
- 置信度
- 图像坐标系下的 `bbox`
- 图像坐标系下的二值 `mask`(尺寸 = bbox.size())
- 像素重心 `center_px`

### 4.4 跨帧追踪

#### F12 像素到物理世界的坐标转换
- 所有识别结果统一转换到**皮带固定坐标系**(单位 mm)。
- 坐标转换必须考虑从**采图时刻** `t_capture` 到**查询时刻** `t_now` 之间,物体跟随皮带产生的位移:

  ```
  Δ_mm = ∫[t_capture, t_now] v_belt(t) dt
  ```

  由系统时间戳直接求差 `t_now - t_capture` 驱动,**不再引入独立的"经验延迟补偿"配置项**(系统调度、推理、传输等延迟已天然包含在 `t_now - t_capture` 内)。
- 每次需要查询物体"当前"物理位置时(关联、派发、阀控下发),都必须以调用时的系统时间重新计算,而不是沿用推理完成时的快照。
- `v_belt(t)` 来源:
  - 优先使用编码器实时速度。
  - 编码器连续读取失败或帧无效时,由 BoardWorker 节流告警;Tracker / Dispatcher 在无有效速度样本时退化为"皮带速度标称值"(F2)。
  - 编码器 raw = 0 表示有效的停止状态,不视为错误。
  - 查询区间内速度变化按最近一次更新的速度近似(阶梯近似),不做更精细的积分。
- 皮带固定坐标系原点在**任务启动瞬间**确定:记录启动瞬间编码器累计读数 `pulse_origin`,**不对硬件清零**;任务期间任意时刻位移 = `(pulse_now - pulse_origin) × 脉冲→mm 系数`;原点选定后任务期间保持不变,直至任务停止并重新启动。

#### F13 跨帧物体追踪与合并

**关联判定**:

- 每个新帧的识别目标与已存在的追踪物体,在**皮带固定坐标系下**计算 mask 相似度(IoU)。
- IoU 阈值可配置;超过阈值视为"同一物体"执行合并,否则视为新物体。
- **多对一冲突**:若单帧内多个新检测对同一已存在追踪物体的 IoU 都超过阈值,采用**贪心最大 IoU** 策略——取 IoU 最大者执行关联,其余进入后续匹配轮次(仍可与其他追踪物体匹配,否则视为新物体)。本期不采用匈牙利全局最优匹配。

**合并规则**:

- **mask**:采用"历史累计融合 mask"。每次关联成功后,将当前检测 mask 与该追踪物体外推到当前时刻的历史 mask 做并集(union)融合,并用融合结果持续更新 `mask`/`bbox`。
- **类别**:维护 `classId → 累计加权面积` 直方图;每次关联,将本次目标面积加到对应 classId 的累计值;最终以累计面积最大的类别作为物体最终类别。
- 几何中心、面积、计数器按融合后的新 mask 重算。

**触发分拣的条件**(满足任一即可):

- **条件 A**:连续 `x` 帧未被任何新识别结果关联上(`x` 可配置,默认 2)。
- **条件 B**:累计被关联更新次数 ≥ `y`(`y` 可配置,默认 10)。

**筛选**:最终类别不在前端勾选的"品类"集合内,直接丢弃,不触发分拣。

**已派发抑制**:

- 物体触发分拣并从追踪列表中移除后,其**累计融合后的 mask**(belt 系)进入"已分拣池",并持续随皮带位移而平移坐标。
- 新帧的检测结果若与已分拣池中任一 mask IoU 超过阈值(沿用关联阈值),该检测**不作为新追踪物体登记、不触发二次分拣**;同时与命中的 ghost mask 做 union 融合并刷新 ghost 时间戳,使抑制池持续更新。
- 抑制池中的 mask,当其整体平移越过分拣线下游一定距离(`dispatched_pool_clear_distance_mm`,可配置,默认 200 mm)后,自动从池中移除。

### 4.5 派发

#### F14.1 机械臂(stub)
- 计算并输出该物体在**机械臂 A 坐标系**与**机械臂 B 坐标系**下的位置。
- **仅日志记录 + 写 `arm_stub.csv`**(按日期滚动),不实际下发 TCP 命令。

#### F14.2 喷阀
- **预计算为主、实时重算为辅**:
  - 物体在触发分拣条件、进入"待喷队列"的瞬间,基于**当前**皮带速度、物体 mask、喷阀位置等,**一次性预计算好**整条开阀序列;不到物体抵达喷阀区再计算。
  - 物体在待喷队列等待期间,若皮带速度变化超过 `speed_recalc_threshold_pct`(默认 20%),**整体取消 + 用新速度重新预计算**整条 pulse 序列;不做 per-pulse 差分。
- **头部留白**:物体刚到达喷阀位置时不立即开阀,需等待物体累计经过喷阀的面积达到 `1/a`(`a` 可配置,默认 8)才开阀,防止气流响应延迟导致喷不准。
- **尾部策略**:**尾部保留喷**——物体后沿完全越过喷阀线后才关阀,尾部不做对称留白。
- **滚动投影追踪**:物体穿越喷阀区间过程中,以 `open_duration_ms` 为步长持续重新投影"喷阀线对应物体哪一行像素 → 横向区间 → 通道集合 → boardId × bitmask"。
- **控制频率**:同 board 上相邻两条 pulse 的最小间隔 = `min_cmd_interval_ms`(默认 50 ms,可配置);间距小于该值的合并(位图 OR、`tClose` 取大)。
- **阀硬件参数**(阀通道总数、每组通道数、横向区间等)均通过**配置文件**定义,**不在前端暴露**。
- **空脉冲处理**:头部留白后已过线 → 不入 pending,告警 `object_too_small`,**不污染 pending 队列**。

### 4.6 配置与持久化

#### F15 统一配置入口
- 所有运行时可调参数集中保存在 `config.ini`(复用现有 `QSettings` 体系)。
- 运行参数、硬件参数、算法阈值与持久化参数均落入同一个 `config.ini`(区分 section)。
- 必须覆盖的字段:见 §3.7。
- 修改 ini 必须重启进程生效;运行中 worker 只用启动时下发的快照,不回读文件。

#### F16 磁盘持久化
- 提供 3 个开关/参数:
  1. **是否保存原图**(总开关)。
  2. **原图采样保存比例**(0.0 ~ 1.0)。
  3. **是否保存识别结果图**(仅当 #1 为真时生效)。
- "保存原图"是对**软件节流后进入推理链的帧**按比例抽样保存,不是相机原始帧。
- "保存识别结果图":只有**该帧的原图被保存**时,才保存对应的识别结果图。
- 保存路径与文件命名:
  - 原图:`save_dir/yyyyMMdd/raw_<hhmmss_zzz>_<seq>.jpg`
  - 识别结果图:与原图同目录,文件名为 `result_<raw_base>.<suffix>`;例如原图 `raw_013015_123_000042.jpg` 对应 `result_raw_013015_123_000042.jpg`。

## 6. 非功能需求

### 6.1 性能(默认配置:fps=2、1024×1024 模型、RK3588 NPU 全核)
- 单帧"采图 → YOLO → 跨帧追踪 → 物理坐标转换"的总耗时必须小于 `1000 / fps ms`,否则按 F10 丢帧;默认配置下 < 500 ms。
- 采图到喷阀命令首次下发的端到端延迟 < 600 ms(不含物体飞行时间)。
- 阀控命令延迟 < 50 ms 可接受。
- 不追求并行推理。
- 连续运行 1 小时内存波动 < 50 MB。
- 默认 fps=2 节奏下,正常情况丢帧计数 = 0。

### 6.2 可靠性
- 运行中相机掉线:不中断任务,仅告警。
- 编码器连续读取失败:使用配置中的"皮带速度标称值"作为 fallback,由 BoardWorker 节流告警。
- 停机后阀门**不允许残留长开**(BoardWorker 必须强制下发关阀帧)。
- 串口打开失败时回退备用设备(`ttyUSB0` → `ttyUSB1`),仍失败则告警冒泡到状态栏。

### 6.3 可维护性
- 所有硬件标定参数、算法阈值、运行时参数必须从配置文件读取,**不得在代码中硬编码**。
- 非显然的物理计算(坐标变换、时间戳位移积分、阀投影计算等)必须有说明注释。
- 跨线程信号传参全部值拷贝,默认 `Qt::QueuedConnection`,禁止跨线程直接函数调用;同一根串口的所有 IO 走单一互斥锁,锁粒度 = 单帧。

### 6.4 兼容性
- 目标平台:RK3588 aarch64 + 海康 MVS SDK + RKNN 2.x + Qt5。
- "颜色 / 标签 / 形态 / 品相" 4 组按钮保留 UI,为后续多维度模型扩展预留接入点。

## 8. 开放项

只列**当前仍未决策 / 仍需现场标定**的事项。代码字段、配置 schema、默认值都已就位,缺的是上板后按真实物料 / 速度 / 气流响应迭代到位的具体数值。

### 8.1 OP-A:算法 / 工艺阈值的现场标定

下表所有项**都已是 `config.ini` 文件可配置项**,分布在 `[pipeline]` 与 `[valve]` 两个 section,由 `loadRuntimeConfig` 在任务启动时一次性读取并下发给各 worker。约束:

- 仅 `config.ini` 可改,**前端 UI 不暴露**,避免操作员误调工艺阈值。
- 修改后**必须重启进程生效**(按 F15,运行中 worker 只用启动时下发的快照)。
- 表中"配置项"列即 `INI section.key`;ini 里未显式给出时回落到 `runtime_config.h` 里的硬编码默认值。

| 配置项 | 默认值 | 物理含义 | 影响 / 现场怎么调 |
|---|---|---|---|
| `pipeline.miss_frames_x` | 2 | 同一追踪物体连续 N 帧未被新检测关联时触发分拣(条件 A) | 过小:物体被遮挡/漏检一次就提前触发,易把同一物分拣两次或位置不准。过大:走出视野后还在等,触发太晚错过喷阀线。需结合 `soft_fps` × 视野长度 / 速度 ≈ 物体在视野中的总帧数确定上限 |
| `pipeline.update_frames_y` | 10 | 累计被关联次数达到 Y 时强制触发(条件 B,作为 miss 兜底) | 防止"物体一直贴着相机不离开,miss 永远到不了"的极端;实际取决于物体在视野中的总帧数,一般 ≈ 视野总帧数的 0.7–0.8 |
| `pipeline.iou_threshold` | 0.30 | belt 系 mask-IoU 关联阈值,也用于已分拣池抑制 | 取决于栅格分辨率(`mask_raster_mm_per_px=2`)+ 速度推算误差 + 物体形变。太低:相邻物体串成同一个;太高:同物体跨帧被拆成两个新追踪 |
| `pipeline.dispatched_pool_clear_distance_mm` | 200 | ghost 越过分拣线下游多少 mm 后从抑制池移除 | 与分拣线下游空间长度、物体可能"被遮挡再出现"的最大距离相关;默认 200 mm 是经验值,过短会让二次分拣抑制失效,过长会无谓占内存 |
| `valve.head_skip_ratio` | 0.125(=1/a, a=8) | 物体经过喷阀线累计面积达 `1/a × totalArea` 才开阀,补偿气流响应延迟 | 取决于:阀气压、阀响应时间、喷头到物体的物理距离、物体在 yb 方向的长度。气压稳/阀快→可减小;反之必须加大 |
| `valve.speed_recalc_threshold_pct` | 20 | 速度变化超过该百分比时,对所有 pending 整体取消并以新速度重新预计算 | 皮带速度稳定 → 可调大避免无谓抖动;速度抖动剧烈 → 必须调小,否则按旧速度算的 pulse 会喷偏 |
| `valve.open_duration_ms` / `valve.min_cmd_interval_ms` | 50 / 50 | 单次开阀时长 + 同板相邻 pulse 最小合并间隔 | 取决于阀机械响应时间下限与物体在 yb 方向的连续覆盖需求;太短气未上来就关,太长会让相邻 pulse 没必要地合并成一坨长开 |

> **现场标定流程**:上板后按"物体类型 × 皮带速度 × 喷阀气压"三个维度做一次现场试跑,把上表逐项调到稳定。这一过程不涉及代码改动,只改 `config.ini` 后**重启进程**即可生效。

### 8.2 OP-B:机械臂 A/B 坐标系标定数值

- **状态**:字段结构已固定为"原点偏移 + 轴方向"四参数 × 两条臂(`a/b_origin_x/y_mm` + `a/b_x/y_sign`),换算公式 = `(belt 系点 − origin) × sign`。
- **何时收敛**:机械臂物理接入(本期是 stub,占位 `0,0,1,1`)。
- **现场怎么定**:在 belt 系上选若干已知点(贴标定块或慢速过 1 颗物体),让机械臂示教到对应位置读出其本体坐标,用 4 个点解出 `origin + sign`。
- **本期能做什么**:Arm 模式下日志和 `arm_stub.csv` 已经按"belt 系坐标 + 占位标定"输出,可以肉眼观察"形状是否合理、左右是否颠倒",作为正式标定前的健全性检查。
