# 光选分拣系统需求文档 v1.1

## 0. 术语约定

- **帧**:除非特别说明,均指**软件节流后实际送入 YOLO 推理的帧**(不是相机原始输出帧)。
- **皮带固定坐标系**:以任务启动瞬间的皮带位置为原点,物体静止在皮带上时坐标保持恒定,单位 mm。
- **最近边缘**:图像坐标中,物体沿皮带运动方向即将离开视野的那条边。

---

## 1. 项目目标与范围

### 1.1 目标

在 RK3588 工控机上构建基于 YOLOv8-Seg 的光学分拣系统,实现"相机采图 → 物体识别 → 跨帧合并 → 驱动分拣器"完整链路。

### 1.2 本期范围

- 包含:相机采图、YOLO 推理、跨帧追踪与合并、坐标转换、喷阀分拣完整闭环、前端参数配置与运行控制、识别结果可视化。
- **不包含**:机械臂实际抓取动作(仅计算"物体在 A/B 机械臂坐标系下的位置"并日志记录,不下发 TCP 命令)。
- **仅 UI 保留**:视觉识别页"颜色 / 标签 / 形态 / 品相"4 组按钮保留可见,不参与业务逻辑。

## 2. 前端交互需求

### F1. 分拣器选择

- 单选:"机械臂" / "喷阀"。
- 选项可用性根据**任务启动前**的连接状态动态决定:
  - 机械臂已建立 TCP 连接 → "机械臂"可选。
  - 喷阀串口打开成功 → "喷阀"可选。
  - 两者都未连接 → 不允许启动任务。

### F2. 运行参数配置

前端暴露以下输入项:

| 参数 | 单位 | 说明 |
|---|---|---|
| 皮带速度(标称值) | m/s | 仅作 fallback,实际流程使用编码器实时值 |
| 相机拍照频率 | 次/秒(fps) | 通过软件节流实现,节流间隔 = 1000 / fps ms |
| 机械臂距图像最近边缘垂直距离 | mm | — |
| 喷阀距图像最近边缘垂直距离 | mm | — |

- 不做输入合理性校验,不合理只告警不拦截。

### F3. 目标品类选择

- 视觉识别页保留"品类"按钮组,选项从配置文件动态加载。
- 按钮与 YOLO 模型 `class_id` 建立映射。
- 勾选的品类集合作为识别结果的**筛选条件**——不在勾选集合内的物体不触发分拣。
- 其它 4 组按钮(颜色/标签/形态/品相)**UI 可见、勾选与否不影响任何业务逻辑**,为后续多维度模型预留。

### F4. 启动/停止任务

- 提供启动与停止控制。
- **启动前强制检查**(失败则不允许启动,提示失败原因):
  1. 相机已连接。
- **启动前软检查**(失败仅告警,仍允许启动):
  1. 所选分拣器已连接。
  2. 编码器能读到有效速度。
  3. 至少勾选 1 个目标品类。
- **运行中**:
  - 所有配置参数(分拣器、速度、fps、两个距离、品类勾选)**只读禁用**。
  - 识别可视化实时更新。
  - 运行中相机掉线不再强制检查,仅告警。
- **启动瞬间"在途物体"策略**:仅处理"启动后第一次进入视野"的物体。任务启动瞬间已经出现在视野中(可能已部分越过镜头)的物体**一律忽略**,不纳入追踪,不触发分拣。
  - 判定方式:任务启动后**第一帧**推理的所有检测结果全部丢弃,不进入追踪。
  - 从第二帧开始,新检测结果按常规流程纳入追踪。
  - 该策略可避免对已部分越过镜头、坐标原点不齐的物体误分拣。
- **停止行为**:硬停——立即终止所有线程、清空追踪列表与"已分拣池"、清空喷阀待喷队列、已预计算但尚未下发的阀控命令一律作废、串口不再发送任何命令。停止后重新启用所有参数控件。

### F5. 实时识别结果可视化

- 画面内容:**仅绘制 YOLO 识别结果的 box、mask、类别标签**。
- 类别标签显示为人类可读的品类名称(对应 F3 中配置的按钮名称),可附带置信度。
- 不绘制 track_id、不绘制历史轨迹。
- 刷新频率 = 识别完成频率(异步推送,不阻塞推理链)。
- **空检测处理**:某帧推理结果为空时,立即清空画面上的 box/mask 覆盖层,只显示原始画面,不保留上一帧覆盖层。

### F6. 状态与告警输出

- 告警同时输出到:**日志文件** + **状态栏**。

### F7. 其它界面约定

- 窗口分辨率与现有布局保持一致,本期不调整。
- 现有"机械臂分段延时标定面板"整块废弃。

## 3. 核心处理流程需求

### F8. 模型常驻

- YOLO 模型在**程序启动时**加载一次、常驻内存。
- **每次推理不重新 init / release**。
- 模型路径通过配置文件指定,**不支持前端配置、不支持热切换**,修改后必须重启进程生效。

### F9. 相机采图与软件节流

- 相机硬件帧率保持现状(60 fps),本期不调整相机侧帧率。
- 仅做软件节流:按 `1000 / fps ms` 的间隔送帧进入推理链。
  - **默认 `fps=2`**(每 500 ms 进入推理一次,每秒 2 张),满足当前业务吞吐需要。
  - `fps` 可通过前端或配置文件调整,上限不超过推理链实测吞吐(见 F10 反压)。
- 节流丢弃的相机原始帧不进入后续任何流程。
- 每张"放行"的帧必须在**相机采图瞬间**打上时间戳(`t_capture`,基于系统单调时钟),该时间戳贯穿后续所有流程,用于计算坐标偏移(见 F12)。

### F10. 帧处理反压机制

- 推理链**单元素队列**(不做并行推理)。
- 新帧到达时若上一帧仍在处理:**丢弃新帧** + 告警(日志 + 状态栏)。
- 丢帧计数需可观测(日志中可见)。

### F11. YOLO 推理输出内容

#### 过滤策略

沿用现有 YOLO 后处理的两步过滤,**两个阈值由硬编码改为配置项**:

- **置信度阈值** `conf_threshold`(默认 0.25):候选框置信度(已取 class score 最大值)低于此值直接丢弃。
- **NMS IoU 阈值** `nms_threshold`(默认 0.45):同类候选框之间 IoU 大于此值的,保留置信度最高者。

过滤后的结果才进入跨帧追踪(F13)。

#### 每个识别目标至少携带

- 类别 `class_id`。
- 置信度。
- 图像坐标系下的 `bbox`。
- 图像坐标系下的二值 `mask`。
- **采图时间戳** `t_capture`(来自 F9,基于系统单调时钟,单位 ms)。
- **推理完成时间戳** `t_infer_done`(基于系统单调时钟,单位 ms)。

采图时间戳必须贯穿整条链路(追踪、派发、分拣控制),任何环节都不得重新赋值。

### F12. 像素到物理世界的坐标转换

- 所有识别结果统一转换到**皮带固定坐标系**(单位 mm)。
- 坐标转换必须考虑从**采图时刻** `t_capture` 到**查询时刻** `t_now` 之间,物体跟随皮带产生的位移:

  ```
  Δ_mm = ∫[t_capture, t_now] v_belt(t) dt
  ```

  由系统时间戳直接求差 `t_now - t_capture` 驱动,**不再引入独立的"经验延迟补偿"配置项**(系统调度、推理、传输等延迟已天然包含在 `t_now - t_capture` 内)。
- 每次需要查询物体"当前"物理位置时(例如追踪关联、派发分拣、阀控命令实际下发),都必须以调用时的系统时间重新计算,而不是沿用推理完成时的快照。
- `v_belt(t)` 的来源:
  - 优先使用编码器实时速度。
  - 编码器无有效值时,退化为"皮带速度标称值"(F2),并写告警日志。
  - 查询区间内速度变化,按最近一次更新的速度近似(阶梯近似),不做更精细的积分。
- 皮带固定坐标系的原点在**任务启动瞬间**确定:
  - 记录启动瞬间的编码器累计读数 `pulse_origin`,**不对硬件清零**。
  - 任务期间任意时刻的皮带位移 = `(pulse_now - pulse_origin) × 脉冲→mm 系数`。
  - 原点选定后任务期间保持不变,直至任务停止并重新启动。

### F13. 跨帧物体追踪与合并

#### 关联判定

- 每个新帧的识别目标与已存在的追踪物体,在**皮带固定坐标系下**计算 mask 相似度(IoU)。
- 相似度超过阈值视为"同一物体",执行合并;否则视为新物体。
- IoU 阈值可配置。
- **多对一冲突处理**:若单帧内多个新检测对同一已存在追踪物体的 IoU 都超过阈值,采用**贪心最大 IoU** 策略——取 IoU 最大者执行关联,其余进入后续匹配轮次(仍可与其他追踪物体匹配,否则视为新物体)。本期不采用匈牙利全局最优匹配。

#### 合并规则

- **mask**:采用"最近一次成功关联的 mask",不做历史融合。
- **类别**:维护"类别 → 累计加权面积"的直方图;每次关联,将新识别目标的 `class_id` 对应的累计值加上本次目标面积;最终分拣时以累计面积最大的类别作为该物体的最终类别。
- 物体的几何中心、面积、计数器等按新 mask 重算。

#### 触发分拣的条件(满足任一即可)

- **条件 A**:物体连续 `x` 帧未被任何新识别结果关联上(`x` 可配置,默认 2)。
- **条件 B**:物体累计被关联更新次数 ≥ `y`(`y` 可配置,默认 10)。

#### 筛选

- 若物体的最终类别不在前端勾选的"品类"集合内,直接丢弃,不触发分拣。

#### 已派发抑制

- 物体触发分拣并从追踪列表中移除后,其**最近一次关联的 mask**(皮带固定坐标系)进入"已分拣池",并持续随皮带位移而平移坐标。
- 新帧的检测结果若与已分拣池中任一 mask IoU 超过阈值(沿用 F13 关联阈值),**直接丢弃**,不作为新追踪物体登记、不触发二次分拣。
- 已分拣池中的 mask,当其整体平移越过视野最远边缘、或越过喷阀/机械臂分拣线下游一定距离(可配置,默认 200 mm)后,自动从池中移除。
- 该机制用于防抖、防遮挡恢复导致的同一物体二次分拣。

### F14. 分拣派发策略

按前端所选分拣器路由:

#### F14.1 机械臂(stub)

- 计算并输出该物体在**机械臂 A 坐标系**与**机械臂 B 坐标系**下的位置。
- **仅日志记录**,不实际下发 TCP 命令。

#### F14.2 喷阀

- **预计算为主、实时重算为辅**:
  - 物体在触发分拣条件、进入"待喷队列"的瞬间,基于**当前**皮带速度、物体 mask、喷阀位置等,**一次性预计算好**整条开阀序列;不到物体抵达喷阀区再计算。
  - 物体在待喷队列等待期间,若皮带速度变化超过阈值,**重新预计算**开阀序列;阈值可配置。
- **头部留白**:物体刚到达喷阀位置时不立即开阀,需等待物体累计经过喷阀的面积达到 `1/a`(`a` 可配置)才开阀,防止气流响应延迟导致喷不准。
- **尾部策略**:**尾部保留喷**——物体后沿完全越过喷阀线后才关阀,尾部不做对称留白。
- **滚动投影追踪**:物体穿越喷阀区间过程中,根据物体当前在喷阀横线上的投影,持续更新应打开的阀通道位图;物体移动会使应开启的阀门发生变化,需要对应地补发新命令。
- **控制频率**:连续两条阀控命令之间的最小间隔 = 当前喷阀开启时长(例如 50 ms),可配置。
- **阀硬件参数**(阀通道总数、每组通道数、每个通道对应的物理横向区间等)均通过**配置文件**定义,**不在前端暴露**。

## 4. 数据与配置需求

### F15. 统一配置入口

- 所有运行时可调参数集中保存在 `config.ini`(复用现有 `QSettings` 体系)。
- 无论是前端输入的运行参数,还是硬件/算法标定参数,均落入同一个 `config.ini`(区分 section)。
- 文件中需至少覆盖:
  - 相机相关(IP、硬件帧率等)。
  - 模型路径。
  - **YOLO 过滤阈值**(`conf_threshold`、`nms_threshold`)。
  - 跨帧追踪阈值(IoU、x、y)。
  - **已分拣池移除距离**(F13)。
  - 喷阀参数(总通道数、分组数、`a`、最小控制间隔、速度重算阈值)。
  - 编码器脉冲→mm 换算系数。
  - 机械臂坐标系标定参数。
  - 磁盘持久化开关(见 F16)。
  - 品类按钮 → `class_id` 映射。
- 具体字段命名与分组在技术方案阶段敲定。

### F16. 磁盘持久化

- 提供 3 个开关/参数:
  1. **是否保存原图**(总开关)。
  2. **原图采样保存比例**(0.0 ~ 1.0)。
  3. **是否保存识别结果图**(仅当 #1 为真时生效)。
- "保存原图"是对**软件节流后进入推理链的帧**按比例抽样保存,不是相机原始帧。
- "保存识别结果图":只有**该帧的原图被保存**时,才保存对应的识别结果图。
- 保存路径与文件命名沿用现有本地保存逻辑。

## 5. 非功能需求

### 5.1 性能

- **帧处理延迟约束**:单帧"采图→YOLO→跨帧追踪→物理坐标转换"的总耗时必须小于 `1000 / fps ms`,否则按 F10 丢帧。
- 不追求并行推理。
- 不追求严格实时;阀控命令延迟 < 50 ms 可接受。

### 5.2 可靠性

- 运行中相机掉线:不中断任务,仅告警。
- 编码器读取失败:使用前端"皮带速度标称值"作为 fallback,写告警日志。
- 连续运行 1 小时无显著内存增长。

### 5.3 可维护性

- 所有硬件标定参数、算法阈值、运行时参数必须从配置文件读取,**不得在代码中硬编码**。
- 非显然的物理计算(坐标变换、时间戳位移积分、阀投影计算等)必须有说明注释。

### 5.4 兼容性

- 目标平台:RK3588 aarch64 + 海康 MVS SDK + RKNN 2.x + Qt5。
- "颜色/标签/形态/品相" 4 组按钮保留 UI,为后续多维度模型扩展预留接入点。

## 7. 验收标准

### 7.1 功能验收

| 编号 | 场景 | 预期 |
|---|---|---|
| AC1 | 相机未接,点击启动 | 提示失败,任务不启动 |
| AC2 | 相机已接、喷阀串口 OK、勾选至少一个品类,点击启动 | 任务正常运行,状态栏无告警 |
| AC3 | 运行中某帧处理超时 | 下一帧被丢弃 + 状态栏告警 + 日志 |
| AC4 | 两个外形相近的目标在视野中依次出现 | 生成两个独立追踪物体,分别触发分拣 |
| AC5 | 同一物体在视野中连续出现并被每帧关联 | 只触发 1 次分拣(且触发时机满足 miss 或 update 任一条件) |
| AC6 | 物体走出视野连续 `x` 帧未再被识别 | 触发分拣 |
| AC7 | 物体在视野中累计关联更新达到 `y` 次 | 强制触发分拣 |
| AC8 | 物体在待喷队列中时皮带速度变化超过阈值 | 重新预计算开阀序列 |
| AC9 | 物体最终类别不在勾选品类内 | 不触发任何分拣动作 |
| AC10 | 切换分拣器为机械臂 | 物体触发时日志打印其在 A/B 臂坐标系下位置,不下发 TCP |
| AC11 | 开启保存原图且采样比例为 0.1 | 约 10% 的推理帧被保存到磁盘 |
| AC12 | 保存原图=false、保存结果图=true | 两种图都不保存 |
| AC13 | 运行中尝试修改参数输入框 | 控件禁用,无法修改 |
| AC14 | 停止任务 | 追踪列表、已分拣池、待喷队列全部清空,串口不再发命令,所有参数控件恢复可用 |
| AC15 | 任务启动瞬间视野中已存在一个物体 | 该物体被任务启动后第一帧丢弃,不触发分拣;之后新进入的物体正常处理 |
| AC16 | 同一帧检测到两个新物体都与同一个追踪物体 IoU 高于阈值 | 贪心选 IoU 最大者关联,另一个作为新物体入轨 |
| AC17 | 物体刚派发分拣后被再次 YOLO 识别(遮挡恢复) | 新检测与"已分拣池"IoU 命中 → 丢弃,不二次分拣 |
| AC18 | 某一帧 YOLO 无任何检测结果 | 画面覆盖层立即清空 |
| AC19 | 配置 `conf_threshold` = 0.9 | 大量低置信度检测被 YOLO 阶段过滤,不进入追踪 |

### 7.2 性能验收

- 默认配置(fps=2、1024×1024 模型、RK3588 NPU 全核)下:
  - 单帧"采图→YOLO→追踪→坐标转换"总耗时 < 500 ms(即节流间隔内完成)。
  - 采图到喷阀命令首次下发的端到端延迟 < 600 ms(不含物体飞行时间)。
  - 连续运行 1 小时内存波动 < 50 MB。
  - 连续运行 1 小时丢帧计数 = 0(正常情况下,2 fps 节奏不触发反压)。

## 8. 开放项(延后决策)
